# 关于时区国际化的理解
1. 用户手机回默认本地的时区，比如北京的用户默认为东八区
2. 医生信息中携带当前的时区信息，当用户进行预约的时候，需要根据时区信息和当前手机时区，进行时间大小的比对，可以是时间戳或者UTC时间，设置一个可以预约的条件（比如当前时间30分钟以前的可以预约，过时的只能查看）， 
3. 在【Doctor Profiles】预约时间的slot需要根据 手机时区的信息，和医生的预约时间需要计算一个差值，保存为医生所在时区的时间戳（UTC时间）比如 Australia/Sydney  为 UTC +11 ， 相差3个小时，医生的时间 8:00AM 相当于 5:00AM （北京时间） 那我们保存的预约日期需要减去3个小时的差值
3. 在【My Appointments】中，显示的时间是否到期，需要根据用户预约医生的时间信息，进行对比，可以是时间戳或者UTC时间，如果预约时间（UTC）小于当前时间，就可以取消，如果到了预约时间（UTC）大于当前时间，那么不可取消。
